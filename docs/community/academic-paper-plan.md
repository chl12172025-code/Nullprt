# Nullprt Academic Paper Plan

## Target Venues
- PLDI (language design + optimization)
- CGO (code generation and optimization)
- ASE / ICSE SEIP (tooling engineering)
- Journal fallback: Software: Practice and Experience

## Paper Structure
1. Motivation and problem framing
2. Language and toolchain architecture
3. Key techniques (compiler, LSP, debugger, profiler)
4. Evaluation methodology
5. Results and ablation studies
6. Threats to validity
7. Related work
8. Conclusion and future work

## Experimental Data Preparation
- Standard benchmarks + real-world project corpus.
- Reproducible environment snapshots and scripts.
- Store raw metrics, aggregated metrics, and confidence intervals.

## Peer Review Invitations
- Internal review pool: maintainers and external compiler engineers.
- Pre-submission review round at least 3 weeks before deadline.
- Track reviewer action items in a dedicated milestone issue.
