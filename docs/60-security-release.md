## 发行版自保护、授权与合规边界

本章将“保护合法软件的知识产权与授权验证”定义为**自保护（self-protection）**能力，并明确与禁止项的边界。所有能力必须满足：**仅作用于本程序/本进程/本文件**（除非用户在受控测试环境中显式授权）。

### 1. 发行版保护流水线（Release Protection Pipeline）
编译器在 `release` profile 下可启用一组可配置变换（默认按安全与兼容性取平衡）：
- **IR 级变换**（推荐作为主要层）：\n  - 控制流扁平化（CFG flattening）\n  - 等价指令替换\n  - 不透明谓词插入\n  - 常量/字符串加固（可选）\n- **二进制后处理**（可选，平台差异大）：\n  - section 布局调整\n  - 符号剥离与重定位策略\n
> 约束：变换必须保持语义等价；必须可通过开关关闭以支持调试/兼容性。

#### 决策清单
- 保护变换的稳定性等级：哪些变换允许默认开启，哪些必须 opt-in？
- 可重复构建：保护变换是否允许引入随机性？若允许，如何提供 deterministic 模式？

### 2. 完整性校验（Integrity）
- **启动校验**：对关键代码段/资源进行哈希校验，检测被篡改。
- **运行时抽样校验**：周期性或事件触发校验（MVP 可先启动校验）。
- **响应策略**：拒绝执行/安全退出/擦除敏感内存（仅限本进程内存）。

### 3. 反调试（Anti-debugging，合规限定）
允许的范围：检测本进程是否被调试（如 `IsDebuggerPresent` 等），用于**授权保护与反动态分析**。\n
禁止的范围：对抗系统安全审计、隐藏进程/连接、绕过安全机制、攻击调试器或其他进程。

### 4. 授权与硬件绑定（合法用途）
- **授权通信**：标准 HTTP/HTTPS 客户端 API（激活、校验、心跳）。\n
- **硬件信号读取（只读）**：磁盘序列、MAC、主板 UUID、BIOS UUID、TPM 序列等。\n
- **绑定策略**：`strict/relaxed`，允许信号丢失容错（例如更换网卡）。\n

#### 决策清单
- 隐私与合规：硬件信号如何最小化采集？是否必须哈希化/盐化后上报？\n
- 离线授权：挑战码/响应码协议是否进入 MVP？\n

### 5. 发行版 vs 开发者版权限差异（规范化）
- **发行版允许**：自保护、授权、硬件信号只读、网络通信、加密 API。\n
- **发行版禁止**：跨进程注入、跨进程内存读写/修改、修改硬件标识、伪装虚拟机、绕过 OS 安全、隐藏行为、远控/外传。\n
- **开发者版额外允许**：调试与分析（断点/性能/日志），以及“受控测试接口”——必须：\n  - 仅在显式启用的测试 profile 下可用\n  - 需要本地许可配置或签名 token\n  - 默认构建不包含或默认关闭\n

### 6. 明确禁止项（写入工具链与 API 设计）
工具链与标准库**不得**提供：\n
- 未授权硬件标识修改或欺骗读取\n
- 伪装虚拟机环境以绕过授权\n
- 跨进程注入/加载模块用于恶意目的\n
- 读取/修改不属于自身进程的内存用于恶意目的\n
- 绕过 OS 安全机制/UAC\n
- 对抗安全审计取证/杀软扫描\n
- 隐藏自身进程/文件/网络连接\n
- 未经同意的监控与数据外传\n
