## Nullprt（NPRT）规格文档总览

本目录定义 Nullprt（NPRT）语言与工具链的**可实现**规格（Spec）与合规边界，用于指导后续 MVP → Beta → 自举（self-hosting）的工程落地。

### 术语
- **NPRT**：Nullprt 语言简称。
- **aegc**：NPRT 编译器可执行文件名称。
- **npkg**：NPRT 包管理器（含仓库/缓存/锁定）。
- **profile**：构建配置（如 `dev`、`release`）。
- **target**：编译目标三元组（示例：`x86_64-windows-msvc`、`aarch64-apple-darwin`、`x86_64-unknown-linux-gnu`）。
- **发行版（distribution build）**：面向最终用户分发的产物（启用自保护；不包含开发者调试与越权能力）。
- **开发者版（developer toolchain）**：完整工具链（包含调试与分析能力，但仍遵循合规边界）。

### 目标（必须满足）
- **静态编译**：生成原生机器码；支持 **x86_64/ARM64**。
- **跨平台**：Windows / macOS / Linux 同一份源代码可编译运行（条件编译允许存在，但不要求改动业务逻辑）。
- **工具链独立**：最终交付物不依赖任何现有语言的运行时/标准库/工具链。
- **向下兼容**：以**标准 ABI** 与现有生态互操作（以 C ABI 为最低公共接口）。
- **合规**：所有“自保护/授权”能力限定在**自进程/自文件/开发者合法测试**边界内；明确禁止项不在设计范围。

### 非目标（明确不做）
- 任何形式的**未授权跨进程注入**、**绕过 OS 安全机制**、**隐藏进程/文件/网络**、**对抗取证/杀软**。
- 面向第三方软件的**未授权内存读写/修改**与**未授权篡改**。

### 版本与稳定性策略（建议）
- **语言版本（`language_version`）**：`0.y.z` 期间允许破坏性变更；`1.0` 后遵循 SemVer。
- **配置格式版本（`config_version`）**：独立版本号；解析器支持至少 N-1 版本向后兼容。
- **标准库版本**：与工具链版本绑定（同发布周期）。
- **仓库协议版本**：独立版本号（便于私有仓库兼容）。

### 文档结构
- `10-language.md`：语法/语义/类型/宏/并发/错误模型/条件编译。
- `20-abi-ffi.md`：C ABI、布局、链接、跨语言互操作边界。
- `30-config-format.md`：`.nprtcfg/.nullprtconfig` 与 `.npkg/.nprtpkg` 格式与 EBNF。
- `40-toolchain.md`：`aegc`、`npkg`、构建系统、格式化、静态分析、文档工具、LSP/DAP。
- `50-runtime-stdlib.md`：运行时与标准库分层、平台抽象层（PAL）。
- `60-security-release.md`：发行版自保护流水线、授权、签名、允许/禁止清单。

### 关键设计决策清单（本章）
- **目标三元组规范**：是否完全兼容 LLVM target triple 命名？（建议兼容，便于工具整合）
- **构建 profile 集合**：`dev/release/test/bench` 是否都要进入 MVP？
- **发行版 vs 开发者版差异**：哪些能力通过编译期开关屏蔽，哪些通过链接剔除，哪些由运行时拒绝？
