## 运行时与标准库分层（最小可行实现）

目标：在不依赖外部语言运行时的前提下，提供 NPRT 程序所需的最小运行时与标准库，并为跨平台实现提供统一抽象层。

### 1. 分层原则
- **core**：无分配、无 OS 依赖（基本类型、traits/interfaces、算术、`Result`）。
- **alloc**：内存分配抽象（全局分配器接口、容器基础）。
- **rt（runtime）**：启动/退出、panic/abort、栈/线程初始化、TLS、异常帧策略（若有）。
- **sys（PAL）**：平台抽象层（Windows/macOS/Linux 的系统调用封装）。
- **std**：在 `core+alloc+sys` 之上提供常用 API（FS、net、time、thread、sync…）。

### 2. 最小运行时（MVP）
- 入口与启动：`main` 调用约定、参数传递（argv/argc）、退出码。
- 崩溃策略：`panic`（开发）与 `abort`（发行）策略可配置。
- 全局分配器：默认实现（MVP 可先用系统堆分配；最终可替换为自研分配器）。

#### 决策清单
- 是否强制可替换分配器 ABI（插件式）？
- 线程栈大小与 TLS 的默认策略。

### 3. 标准库最小集（MVP）
- **内存/容器**：`Vec`、`String`、`Box`（或等价），哈希表可延后。
- **文件系统**：打开/读写/遍历目录、路径处理。
- **时间**：单调时钟、系统时间、sleep。
- **线程与同步**：`Thread`、`Mutex`、`RwLock`、`Condvar`、`Atomic`（最小集合）。
- **通道**：多生产者/多消费者 channel（MVP 可先单一实现）。
- **环境与进程**：读取环境变量、当前目录、进程退出。
- **网络（可分阶段）**：MVP 可先 TCP/UDP；HTTP/TLS 放到 Beta（或作为可选包）。

### 4. 发行版 vs 开发者版 API 权限映射（本章占位）
发行版允许：
- 基础系统、网络、加密、安全（自保护）、硬件信息只读（用于绑定）、授权通信。

发行版禁止：
- 任何会修改硬件标识/伪装虚拟机/跨进程注入/越权调试的 API。

开发者版额外允许：
- 调试与分析 API（断点、栈回溯、性能计数器、日志）。
- 受控测试接口（必须显式标注为 `dev_only`，且默认关闭，需要测试许可配置）。

### 5. 平台抽象层（PAL）建议接口
- `sys.mem`：虚拟内存申请/保护（`mmap/VirtualAlloc`）
- `sys.fs`：文件句柄、权限、符号链接（差异需封装）
- `sys.net`：socket、poll/epoll/kqueue、DNS
- `sys.crypto`：若自研 crypto，尽量不直接绑定 OS；若使用 OS 原语需明确可替代层

#### 决策清单
- Windows 的 IO 模型：IOCP vs select/poll 抽象如何对齐？
- TLS/SSL：自研实现 vs 封装系统库（独立性要求下建议自研或随工具链分发的自有实现）。
