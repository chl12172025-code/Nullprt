## 工具链规格（aegc / npkg / build / fmt / lint / doc / LSP/DAP）

本章给出工具链模块拆分与对外接口。MVP 以“能跑通端到端”为第一目标：配置解析 → 依赖解析 → 编译 → 链接 → 运行（可选测试）。

### 1. 编译器 `aegc`
#### 1.1 阶段划分
- **前端**：词法/语法 → AST；宏展开；名称解析；类型检查；借用检查（MVP 最小规则）。
- **中端**：生成 NPRT-IR；优化（按 opt_level）；插桩/保护（release）。
- **后端**：目标代码生成（x86_64/ARM64）；对象文件；链接驱动。

#### 1.2 编译缓存与增量
- 编译单元（module/package）产物缓存：基于源码 hash + 配置 + 依赖图。
- 目标：MVP 可先实现“模块级缓存”，增量重建晚些阶段补齐。

#### 1.3 构建模式
- `dev`：保留调试信息；关闭或弱化保护变换；启用诊断友好错误。
- `release`：启用保护流水线（见 `60-security-release.md`）；可剥离符号；开启更高优化。

### 2. 包管理器 `npkg`
#### 2.1 主要职责
- 解析 `deps.npkg` 与项目 `NPRTcfg` 的依赖声明。
- 语义化版本解析（包含 `^`/`~`/范围语法需定义，MVP 可先精确版本 + `>=,<`）。
- 下载、校验（hash/签名）、缓存、离线构建。
- 生成锁文件（可重现构建）。

#### 2.2 仓库与镜像
- 支持中央仓库与私有仓库；协议见 `spec/npkg-registry-protocol.md`（由本仓库提供）。

### 3. 构建系统（声明式）
建议提供 `aegc build` 的统一入口，读取配置文件后执行：
- 依赖解析（npkg）
- 编译（aegc）
- 链接（aegc link driver）
- 产物输出到 `build/` 目录

#### 决策清单
- 是否提供独立构建工具 `nprtbuild`，还是 `aegc` 内置子命令？

### 4. 格式化器 / 静态分析 / 文档生成
- `nprtfmt`：统一格式化（MVP 可只支持配置/少量语法子集）。
- `nprtlint`：未使用变量/函数、潜在内存错误模式（依赖类型与借用信息）。
- `nprtdoc`：从 doc 注释生成 API 文档（MVP 可先生成符号索引）。

### 5. IDE 支持：LSP / DAP
#### 5.1 LSP（MVP）
- 解析与诊断（语法错误、名称解析错误）
- 跳转到定义、查找引用（基于符号表）
- 补全（标识符、成员、导入）
- 格式化（可先做文件级基本格式化）

#### 5.2 DAP（开发者版优先）
- MVP 可先不实现完整 DAP；先保证编译器能产出可被系统调试器识别的调试信息（PDB/DWARF）。

### 6. 安装与版本管理（概述，细节见 Stage0/Stage2）
- 安装包：Win `.exe` / macOS `.pkg` / Linux `.deb/.rpm/.tar.gz`。
- 环境变量：以 `NPRT_` 前缀，包括根目录、bin 路径、cache、toolchains 目录等。
- 版本管理工具：列出/安装/卸载/切换/项目锁定版本。
