## 语言规格（MVP → Beta）

本章定义 NPRT 的语法与语义范围。为保证可落地，**先定义 MVP 子集**，再逐步扩展到完整愿景特性。

### 1. 源码组织与命名
- **包（package）**：发布与依赖的基本单位。
- **模块（module）**：包内源码的命名空间单位。
- **入口**：可执行包必须提供 `main`。

#### 决策清单
- 包与模块的映射：文件系统路径到模块名的规则（是否允许 `mod.nprt` 风格聚合？）
- 名称解析：是否区分 `package::module::item` 与 `module::item`？

### 2. 类型系统（强类型 + 推断边界）
- **基础类型**：`i{8,16,32,64}`, `u{8,16,32,64}`, `isize/usize`, `bool`, `char32`, `f{32,64}`（若引入）。
- **复合类型**：`struct`、`enum`（代数数据类型）、数组、切片、指针类型（受限）。
- **类型推断**：局部推断；公开 API 需要显式类型（建议）。
- **泛型**：编译期单态化（MVP 可先支持“函数泛型”，类型泛型延后）。

#### 决策清单
- `char` 的语义：Unicode scalar vs UTF-8 byte。
- 浮点策略：MVP 是否必须支持 `f32/f64`？
- `enum` 表示：tag + payload 布局是否固定/可控（与 `repr` 属性相关）。

### 3. 所有权与借用（无 GC 的内存安全）
目标：在不引入垃圾回收的情况下提供内存安全。

#### MVP 最小规则（建议）
- **唯一所有者**：值默认拥有其资源；移动语义。
- **借用**：`&T` 共享借用，`&mut T` 独占借用；同一作用域内满足互斥。
- **生命周期**：MVP 可先采用“作用域生命周期 + 非逃逸借用”限制，逐步扩展到显式生命周期参数。

#### 决策清单
- 是否允许裸指针 `*const/*mut`：仅在 `unsafe` 块内？
- `unsafe` 的范围与语义：哪些操作需要 `unsafe`？

### 4. 错误模型（无异常）
- 使用 `Result<T, E>` 或等价机制进行错误传播。
- 提供 `?` 风格语法糖（可选）用于“遇错返回”。

#### 决策清单
- 错误类型 `E` 的约束：必须实现某个 trait/interface？
- `panic`/`abort`：仅用于不可恢复错误？其与 `Result` 的边界。

### 5. 模式匹配
- `match` 支持对 `enum` 与结构体解构。
- 穷尽性检查：MVP 至少对 `enum` 做穷尽性。

#### 决策清单
- `match` 的守卫（guard）是否进入 MVP？
- 绑定语法：`x @ pattern` 是否需要？

### 6. 宏系统（编译期代码生成）
为确保独立工具链可落地：先定义可实现的分层。
- **MVP**：声明式 hygienic 宏（类似 token-tree 变换）。
- **Beta**：过程宏（需要稳定的 AST/IR 接口）。

#### 决策清单
- token 模型：是否自研 token-tree（推荐）以降低语法耦合？
- 宏展开阶段：在解析前/解析后/类型检查前？

### 7. 并发与异步
愿景包含消息传递并发与受约束共享内存并发。
- **MVP**：线程 + channel（消息队列）+ `atomic/mutex` 最小集。
- **Beta**：协程/异步任务、运行时调度器。

#### 决策清单
- 异步模型：`async/await` 语法是否进入 MVP？
- `Send/Sync` 类标记 trait 是否必须？

### 8. 属性（attributes）与条件编译
- 属性控制：内存布局、ABI、平台特定行为、链接指令。
- 条件编译：按 `target_os/arch/profile/feature`。

#### 决策清单
- 属性语法：`@attr(...)` 还是 `#[attr(...)]`（需“独立语法体系”，避免与既有语言混淆）。
- 条件编译表达式语言：是否支持布尔表达式（`and/or/not`）？
