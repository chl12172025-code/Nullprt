# Nullprt 1.0 发布与质量保证

本文定义稳定版 1.0 的发布治理、第三方安全审计、合规审查与持续质量保证要求。

## 1.0 发布治理
- 发布清单完成度检查：必须通过 `release_checklist.json` 全项校验（100%）。
- 发布冻结期：`RC-14` 至 `GA+0` 禁止引入非修复型变更。
- 候选版本测试周期：RC 至少运行 14 天，包括功能、性能、稳定性、回归测试。
- 回滚应急预案：保留上一稳定版可回滚工件，要求 30 分钟内可切换。

## 第三方安全审计
- 审计范围：编译器/运行时/工具链/发布工件供应链路径。
- 审计机构选择标准：资质、独立性、语言与编译器审计经验、披露流程能力。
- 结果公开程度：高危问题与修复状态公开；敏感细节按责任披露策略处理。
- 修复计划：按严重级别给出修复 SLA（Critical 7d、High 14d、Medium 30d）。

## 合规审查
- 出口管制分类：提供分类结论与适用国家/地区限制映射。
- 加密算法出口许可：记录许可要求、备案状态与版本变更影响。
- 开源许可证兼容性：建立依赖许可证兼容矩阵并阻断冲突依赖。
- 隐私政策声明：明确遥测、日志、数据保留与用户删除策略。

## CI 质量门禁
- 性能回归测试：基线比较 + 噪声过滤 + 统计检验。
- 安全漏洞扫描：SAST + 产物扫描 + 规则更新。
- 依赖项漏洞检查：锁定版本清单与 CVE 例行更新。
- 覆盖率门禁：核心模块覆盖率阈值（语句/分支）不低于策略值。

## 测试覆盖扩展
- 模糊测试（Fuzzing）
- 符号执行测试
- 并发压力测试
- 长时间稳定性测试（Soak）

## 文档质量门禁
- 拼写检查（中英文）
- 链接有效性检查
- 示例代码编译验证
- 中英文版本一致性检查
